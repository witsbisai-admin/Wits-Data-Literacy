1. Platform Overview & Architecture

The platform is a single self-contained HTML file (~280 KB). It contains all CSS, JavaScript, content, lab data, quizzes, and the certificate generator — with no external dependencies except Google Fonts (which gracefully degrade if unavailable). There is no database, no server-side code, and no build step.

LayerTechnologyNotes
FrontendVanilla HTML5 + CSS3 + JavaScript (ES6+)No frameworks. No npm. No build step.
StylingCSS custom properties + Google Fonts (Sora, Lora, JetBrains Mono)Fonts load from CDN; fall back to system fonts if blocked.
Data persistencelocalStorage + sessionStoragePer-user, per-browser. No cross-device sync without a backend.
AuthClient-side accounts stored in localStorageSuitable for internal/trusted environments. Not cryptographic.
Web servernginx (recommended) or any static file serverEven Python's http.server works for local testing.
⚡ Key Insight

Because the entire platform is one HTML file, deployment is simply: put index.html somewhere that serves it over HTTP/HTTPS. Every option below does exactly this, with varying degrees of production hardening.

4. Option A — Docker (Recommended)

🐳

Docker Deployment

Best for: cloud VMs, university servers, consistent reproducible environments · ~5 min setup

Tools Required

ToolVersionInstall CommandRequired
Docker Engine24.x+curl -fsSL https://get.docker.com | bashRequired
Docker Composev2.x+apt install docker-compose-pluginRequired
CertbotLatestsnap install --classic certbotHTTPS only
1

Install Docker on your server

# One-line install (official Docker script)
curl -fsSL https://get.docker.com | bash

# Add your user to docker group (avoids needing sudo)
sudo usermod -aG docker $USER && newgrp docker

# Verify
docker --version          # Docker version 24.x.x
docker compose version    # Docker Compose version v2.x.x
2

Upload the package to your server

# From your local machine via SCP:
scp -r ./wits-complete-package  user@YOUR-SERVER-IP:/home/user/

# Or clone from a git repo:
git clone https://github.com/your-org/wits-deploy.git

# Or SFTP using FileZilla / WinSCP / Cyberduck
3

Set your domain in nginx/default.conf

# Open the config
nano nginx/default.conf

# Find and change:
server_name localhost;

# To your actual domain (examples):
server_name datalit.wits.ac.za;
server_name biso-platform.wits.ac.za;
4

Build and start the container

cd wits-complete-package/docker

# Build image and start container in background
docker compose up -d --build

# Check it's running
docker compose ps
# Expected:  wits_platform   Up 30 seconds   0.0.0.0:80->80/tcp

# Verify health endpoint
curl http://localhost/health
# Expected: healthy
5

Open firewall ports

sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw status

# On AWS/Azure/GCP: also open ports in your
# Security Group / Firewall Rules in the cloud console
✅ Success Indicators
  • docker compose ps shows Up with health healthy
  • curl http://localhost/health returns healthy
  • Browser at http://your-server-ip shows the Wits sign-in screen

Useful Docker Commands

# View live logs
docker logs -f wits_platform

# Stop
docker compose down

# Restart after config change
docker compose restart platform

# Update platform HTML (zero downtime)
docker cp platform/index.html wits_platform:/usr/share/nginx/html/index.html
docker exec wits_platform nginx -s reload

# Full rebuild after Dockerfile change
docker compose up -d --build

5. Option B — Bare nginx on Ubuntu/Debian

🐧

Bare nginx on Ubuntu/Debian

Best for: existing Ubuntu servers without Docker, traditional sysadmin setups

1

Install nginx

sudo apt update && sudo apt install nginx -y
sudo systemctl enable nginx
sudo systemctl start nginx
nginx -v   # nginx version: nginx/1.18.0 (or newer)
2

Copy platform file to web root

sudo mkdir -p /var/www/wits-platform
sudo cp platform/index.html /var/www/wits-platform/index.html
sudo chown -R www-data:www-data /var/www/wits-platform
3

Install nginx configuration

sudo cp nginx/nginx.conf /etc/nginx/nginx.conf

# Copy virtual host config with correct root path
sudo sed 's|root  /usr/share/nginx/html;|root /var/www/wits-platform;|g' \
    nginx/default.conf > /etc/nginx/sites-available/wits-platform

# Set your domain
sudo nano /etc/nginx/sites-available/wits-platform
# Change: server_name localhost;  →  server_name your-domain.wits.ac.za;

# Enable site, disable default
sudo ln -sf /etc/nginx/sites-available/wits-platform /etc/nginx/sites-enabled/
sudo rm -f /etc/nginx/sites-enabled/default

# Test and reload
sudo nginx -t && sudo systemctl reload nginx
4

Open firewall

sudo ufw allow 'Nginx Full'
sudo ufw enable
sudo ufw status
5

Install systemd service (optional — for auto-start on reboot)

sudo cp nginx/wits-platform.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable wits-platform

Useful nginx Commands

sudo nginx -t               # Test config syntax
sudo nginx -s reload         # Reload config without downtime
sudo systemctl restart nginx # Full restart
sudo systemctl status nginx  # Check status
sudo tail -f /var/log/nginx/access.log   # Live access log
sudo tail -f /var/log/nginx/error.log    # Live error log

6. Option C — Static Hosting

☁️

Static Hosting (Netlify / Vercel / GitHub Pages)

Best for: fastest setup, zero server management, free tier available · 2–5 min

⚡ Why This Works

The platform is entirely self-contained — no server-side logic, no database calls. Any static file host can serve it. All authentication and data storage happen in the learner's browser (localStorage).

Netlify (Easiest — Recommended)

1

Create free account at netlify.com

Sign up with GitHub, GitLab, email, or Google. No credit card needed.

2

Create new site → Deploy manually

Dashboard → Add new site → Deploy manually.

3

Drag and drop the platform folder

Drag the entire platform/ folder (containing index.html) onto the Netlify drop zone. Do not drag the file directly — drag the folder.

4

Get your URL

Netlify instantly provides a URL like random-name.netlify.app. The platform is live.

5

Add custom domain (optional)

# In Netlify: Site Settings → Domain Management → Add custom domain
# Enter: datalit.wits.ac.za
# Follow DNS instructions → HTTPS is configured automatically
6

For CI/CD auto-deploy on git push:

# Install Netlify CLI
npm install -g netlify-cli

# Link repo and deploy
netlify link
netlify deploy --prod --dir ./platform

Vercel

# Install Vercel CLI
npm install -g vercel

# Deploy from the project root (vercel.json tells it where index.html is)
vercel --prod

# Or via dashboard: vercel.com → New Project → Import → deploy

GitHub Pages

# 1. Create a GitHub repository and push this project
git init && git add . && git commit -m "Initial deploy"
git remote add origin https://github.com/YOUR-ORG/wits-datalit.git
git push -u origin main

# 2. In GitHub: Settings → Pages → Source → GitHub Actions
# 3. The workflow file (.github/workflows/deploy.yml) runs automatically
# 4. Platform available at: https://YOUR-ORG.github.io/wits-datalit/

7. Option D — Canvas LMS Integration

🎓

Canvas LMS Integration

Best for: institutions using Canvas as their primary LMS — embeds platform directly in a Canvas course

Three Methods
  • Method 1 — Redirect Tool: links to hosted platform in a new tab. 5 min, no IT needed.
  • Method 2 — iFrame Embed: embeds platform inline in a Canvas Page. 10 min, may need domain whitelist.
  • Method 3 — LTI 1.3: full grade passback to Canvas gradebook. Requires backend server + Canvas admin.

Method 1 — Redirect Tool (5 minutes, no IT needed)

1

First deploy the platform using any of Options A, B, or C to get a URL

You need a live URL such as https://wits-datalit.netlify.app or https://datalit.wits.ac.za.

2

In Canvas: Course → Settings → Apps → View App Configurations → + App

3

Select "By URL" configuration type and fill in:

Name:          Data Literacy for Data Champions
Consumer Key:  wits-datalit
Shared Secret: wits2025
Launch URL:    https://YOUR-DEPLOYED-URL.com

Add to Modules: + → External Tool → select the tool → Add Item

Method 3 — LTI 1.3 (Grade Passback to Canvas Gradebook)

LTI 1.3 enables Canvas to pass the learner's verified identity to the platform and receive quiz/completion scores back into the Canvas gradebook automatically.

Requirements

Canvas Admin → Developer Keys → + Developer Key → LTI Key

Configure the LTI Tool with these settings:

Redirect URIs:       https://YOUR-PLATFORM-URL.com/lti/launch

8. SSL / HTTPS Setup (Let's Encrypt — Free)

Before You Start
  • Your domain DNS must point to your server IP (verify: nslookup your-domain.wits.ac.za)
  • Port 80 must be open and the platform must be serving HTTP first
  • Not needed for Option C (Netlify/Vercel provide HTTPS automatically)
# Install Certbot
sudo snap install --classic certbot
sudo ln -sf /snap/bin/certbot /usr/bin/certbot

# Option B (bare nginx) — automatic config update
sudo certbot --nginx -d your-domain.wits.ac.za

# Option A (Docker) — standalone mode (stops container briefly)
cd docker && docker compose down
sudo certbot certonly --standalone -d your-domain.wits.ac.za
# Then uncomment SSL lines in nginx/default.conf and restart
docker compose up -d --build

# Test auto-renewal (certificates renew every 90 days automatically)
sudo certbot renew --dry-run

After getting the certificate, uncomment the HTTPS server block in nginx/default.conf and the HTTP-to-HTTPS redirect. For Docker, also uncomment the Let's Encrypt volume and port 443 in docker-compose.yml.

9. Updating the Platform

When a new version of index.html is released, replace it using one of these methods:

# Option A — Docker (zero-downtime hot swap)
docker cp platform/index.html wits_platform:/usr/share/nginx/html/index.html
docker exec wits_platform nginx -s reload

# Option A — Full rebuild (if Dockerfile changed)
cd docker && docker compose up -d --build

# Option B — Bare nginx
sudo cp platform/index.html /var/www/wits-platform/index.html
sudo nginx -s reload

# Option C — Netlify CLI
netlify deploy --prod --dir ./platform

# Option C — GitHub Pages (auto on push)
git add platform/index.html && git commit -m "Update platform" && git push

# Using the deploy script
bash deploy.sh --update
